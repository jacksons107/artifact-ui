artifact-ui is an MCP tool that turns a JSON 'system_spec' into an interactive, multi-view HTML diagram (Architecture, Layers, Sequences, Code Detail, Matrix, Components). The Architecture view supports toggleable filters (node kind, change status, edge kind, group — deselecting all of one type fades it, with cascading fade to a group's members/edges and a node's own edges) and an animated playback timeline that scrubs through any sequence as a traveling pulse, fading everything unrelated to it. This page documents the tool's own architecture at the system level (below) and, via the Code Detail tab, drills into the key rendering/persistence functions in server.py and the system_spec/ package, and the system_spec JSON format itself — i.e. what an agent actually generates.

stdio JSON-RPC MCP server + a FastAPI app on :8765, sharing one asyncio event loop. Exposes get_example, render_artifact, save_artifact, get_artifact_spec. async def call_tool(name, arguments) -> list[TextContent] # 'render_artifact' branch
rendered_html = renderer(data)
html = inject_runtime(rendered_html, artifact_id)
artifact_dir = ARTIFACT_DIR / artifact_id
artifact_dir.mkdir(parents=True, exist_ok=True)
(artifact_dir / f"{artifact_id}.html").write_text(html, encoding="utf-8")
(artifact_dir / f"{artifact_id}.spec.json").write_text(json.dumps(data, indent=2), encoding="utf-8")

persist_path = arguments.get("path")
if persist_path:
    dest = Path(persist_path)
    dest.parent.mkdir(parents=True, exist_ok=True)
    dest.write_text(rendered_html, encoding="utf-8")
    spec_path_for(dest).write_text(json.dumps(data, indent=2), encoding="utf-8")

⟨⟩Nodeclass

A box in the Architecture diagram. 'kind' picks its icon/color (service, db, queue, external, module, function, class, file, package). Code-level fields (signature, code_snippet, previous_code_snippet, file_path, line_range, status) add the detail-panel snippet, location metadata, and Changes-tab entries.

⟨⟩Edgeclass

An arrow from one node to another. 'kind' sets its color/style (calls, imports, depends, emits, subscribes, reads, writes, deploys, owns, returns, throws, overrides, implements, instantiates). 'async: true' or certain kinds render as dashed. Toggling an edge-kind filter off fades matching edges; toggling a node off (via kind/status filter or its group) also fades that node's connected edges.

⟨⟩Groupclass

A labeled bounding box around a set of node ids. kind='layer' groups also become swim-lanes in the Layers tab. Other kinds (package, team, domain, deployment) are drawn but don't add a Layers lane. Toggling a group off fades its bounding box, member nodes, and any edges touching those members.

⟨⟩Group.detailclass

An optional nested spec on a group: its own 'nodes' and 'edges' (own id namespace, typically function/class/file kinds). Each group with a non-empty detail.nodes gets an entry in the Code Detail tab's module dropdown, rendered as its own mini architecture diagram + detail panels. This Spec Language section is itself a Group.detail.

⟨⟩Sequenceclass

An ordered list of steps (from -> to, with a label) between top-level node ids, rendered as a sequence/swimlane diagram in the Sequences tab and overlaid on the Architecture tab via its Animate control. Both support a scrub/play timeline that walks the steps as a traveling pulse, fading nodes/edges unrelated to the active sequence, with a click-to-reveal panel for any step's example data.
