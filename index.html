◈MCP Clientservice

The MCP client embedded in Claude Code. Maintains a stdio connection to server.py. Serializes tool calls as JSON-RPC and deserializes responses. Presents available tools to Claude at session start.

◈server.pyservice

The heart of the tool. Runs two concurrent servers: (1) an MCP stdio server that receives tool calls from Claude Code, and (2) a FastAPI HTTP server on port 8765 that serves rendered HTML files to the browser. Handles get_example and render_artifact tools.

□template_loader.pymodule

Builds the render_artifact tool description that Claude sees. This is what tells Claude about the spec schema, node/edge/group kinds, and how to invoke the tool. Keeping it accurate and concise is key to good agent behavior.

□system_spec_examples.pymodule

Three reference specs: sys_microservices (e-commerce platform), sys_event_driven (IoT pipeline), sys_monolith (Rails app). Claude calls get_example to read one of these before building a spec, ensuring it understands the correct schema shape.

Tech
Python
Receives From
← server.py reads · get_example
□system_spec.pymodule

The renderer. Takes a validated spec dict and produces a complete HTML page with five views: Architecture (SVG box-and-arrow), Layers (swim-lanes), Sequences (swim-lane diagrams), Matrix (adjacency table), Components (filterable table). Layout uses Kahn's longest-path algorithm for topological ordering.

□design_system.pymodule

CSS design tokens and shared HTML utilities. Ivory/slate/clay/olive/rust palette, serif headings. Imported by system_spec.py for consistent visual style.

⬡/tmp/artifact_uidb

Temporary directory where rendered HTML files are written. Each artifact is a single self-contained HTML file named {artifact_id}.html. Also stores server.pid so the running server process can be tracked.

◇Browserexternal

The user's browser. Opened automatically via webbrowser.open() when render_artifact is called. Fetches the artifact HTML from the FastAPI server and renders the interactive diagram. The Save button downloads the current page as a standalone HTML file.

Calls / Sends
→ server.py calls · GET /artifact/{id}
Receives From

≡CLAUDE.mdfile

Two files: ~/.claude/CLAUDE.md (global) and artifact-ui/CLAUDE.md (project). Loaded into Claude's context at session start. Tell Claude to always call get_example first, always use template='system_spec', prefer render_artifact over AskUserQuestion for decisions.
