artifact-ui is an MCP tool that turns a JSON 'system_spec' into an interactive, multi-view HTML diagram (Architecture, Layers, Sequences, Code Detail, Matrix, Components). The Architecture view's layout and SVG drawing run entirely client-side in arch_engine.js — Python only validates/resolves the spec and emits it as JSON. Groups are collapsible nodes: a group's 'members' ARE its drill-down content (real functions/files/etc), with no separate nested spec to keep in sync — this page is itself built that way, so every group below has its own Code Detail entry made of the real functions it groups. 'clone_of' lets a group's whole member subtree be reused under a new id-prefix. This page documents the tool's own current architecture, including its client-side layout engine and its two dev-only test suites.

⟨⟩Groupclass

A labeled, collapsible bounding box around a set of node and/or group ids. There is no separate 'detail' block — a group's own 'members' ARE what it expands to reveal (see the arch_engine, mcp_server, system_spec, and spec_format groups on this very page: each one's members are the real functions/files they group). Every id may have at most one parent group, which is what makes collapse/expand unambiguous. A group must have either 0 or 2+ members — a single-member group is rejected, since it adds no value over referencing that member directly (this page used to have three of those — agent_layer, mcp_layer, spec_layer — removed once the rule landed). kind='layer' groups also become swim-lanes in the Layers tab.
