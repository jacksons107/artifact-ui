artifact-ui is an MCP tool that turns a JSON 'system_spec' into an interactive, multi-view HTML diagram (Architecture, Layers, Sequences, Code Detail, Matrix, Components). This page documents the tool's own architecture at the system level (below) and, via the Code Detail tab, drills into both the key rendering/persistence functions in server.py and system_spec.py, and the system_spec JSON format itself — i.e. what an agent actually generates.

def render_code_detail_html(spec: dict) -> str
def render_code_detail_html(spec: dict) -> str:
    detail_groups = [g for g in spec["groups"] if g.get("detail", {}).get("nodes")]
    if not detail_groups:
        return ""
    ...
    for g in detail_groups:
        sub = parse_spec({"title": g["label"], "nodes": g["detail"]["nodes"],
                          "edges": g["detail"].get("edges", [])})
        positions = layout_graph(sub["nodes"], sub["edges"], [])
        prefix = f'cd-{g["id"]}-'
        svg    = render_architecture_svg(sub, positions, id_prefix=prefix)
        panels = render_detail_panels(sub, id_prefix=prefix)
        ...

⟨⟩Nodeclass

A box in the Architecture diagram. 'kind' picks its icon/color (service, db, queue, external, module, function, class, file, package). Code-level fields (signature, code_snippet, previous_code_snippet, file_path, line_range, status) add the detail-panel snippet, location metadata, and Changes-tab entries.

An arrow from one node to another. 'kind' sets its color/style (calls, imports, depends, emits, subscribes, reads, writes, deploys, owns, returns, throws, overrides, implements, instantiates). 'async: true' or certain kinds render as dashed.

⟨⟩Group.detailclass

An optional nested spec on a group: its own 'nodes' and 'edges' (own id namespace, typically function/class/file kinds). Each group with a non-empty detail.nodes gets an entry in the Code Detail tab's module dropdown, rendered as its own mini architecture diagram + detail panels. This Spec Language section is itself a Group.detail.
